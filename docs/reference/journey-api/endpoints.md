# Endpoints

Types for describing HTTP operations. Narrative version in the [Writing journeys guide](../../guide/writing-journeys/endpoints).

## `Endpoint<TResponse>`

```ts
type Endpoint<TResponse = unknown> = EndpointRef<TResponse> | EndpointDescriptor;
```

The union `step.endpoint` accepts.

## `EndpointRef<TResponse>`

```ts
interface EndpointRef<TResponse> {
  readonly method: HttpMethod;
  readonly path: string;
  readonly operationId?: string;
  // internal type brand carrying TResponse
}
```

Generated by `@journey/codegen` into `generated/endpoints.ts`. The response type is a phantom brand — erased at runtime.

## `EndpointDescriptor`

```ts
interface EndpointDescriptor {
  readonly method: HttpMethod;
  readonly path: string;
  readonly baseUrl?: string;
}
```

Hand-written escape hatch for operations outside the spec. Response type is `unknown` unless the caller casts.

## `HttpMethod`

```ts
type HttpMethod = "GET" | "POST" | "PUT" | "PATCH" | "DELETE" | "HEAD" | "OPTIONS";
```

## `ResponseOf<E>`

```ts
type ResponseOf<E> = E extends EndpointRef<infer R> ? R : unknown;
```

Extracts the response type from an endpoint. Used by `StepOptions<E>` to type `assert` / `after` callbacks.

## `isEndpointRef(e)`

```ts
function isEndpointRef(e: Endpoint): e is EndpointRef<unknown>;
```

Runtime guard — returns `true` when the endpoint has an `operationId`. Useful for tools that walk journey steps (e.g. the k6 adapter).

```ts
if (isEndpointRef(step.endpoint)) {
  console.log(`${step.endpoint.method} ${step.endpoint.path} (op ${step.endpoint.operationId})`);
} else {
  console.log(
    `${step.endpoint.method} ${step.endpoint.path} on ${step.endpoint.baseUrl ?? "<default>"}`,
  );
}
```
